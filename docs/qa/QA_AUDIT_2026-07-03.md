## 3. Fixes Applied

### Fix 1: `.ts` import extensions (TS5097) — 12 files

**Root cause:** `tsconfig.base.json` has `allowImportingTsExtensions: false`, but test files imported with `from './index.ts'`.

**Files fixed:**
- `packages/@nai/{audit,policy-fga,approval,auth,billing,entitlement,policy-engine,runtime-sdk}/src/test.ts`
- `packages/@nai/auth/middleware/astro.ts`
- `apps/auth/src/{d1-audit-store.ts,index.ts}`

**Fix:** `sed -i '' "s|from './index.ts'|from './index'|g"` — removed `.ts` extension from all relative imports.

### Fix 2: `@nai/email` — `formatAddress` type + DOM lib + `noUncheckedIndexedAccess`

**File:** `packages/@nai/email/src/client.ts`
**Fix:** Changed `formatAddress(addr: EmailAddress)` → `formatAddress(addr: string | EmailAddress)` to accept both normalized strings and EmailAddress objects.

**File:** `packages/@nai/email/tsconfig.json`
**Fix:** Added `"lib": ["ES2022", "DOM"]` — base config only had `ES2022`, missing `console` and `fetch` globals.

**File:** `packages/@nai/email/src/test.ts`
**Fix:** `sed -i '' "s/mock\.sent\[\([0-9]*\)\]\./mock.sent[\1]!./g"` — added non-null assertions for `mock.sent[0]` access (base config has `noUncheckedIndexedAccess: true`).

### Fix 3: `@nai/email` — `createEmailService` not re-exported

**File:** `packages/@nai/email/src/index.ts`
**Fix:** Changed `export { EmailService } from './service'` → `export { EmailService, createEmailService } from './service'`. The function existed in `service.ts` but was not re-exported from the package barrel.

### Fix 4: `@nai/entitlement` — type narrowing (TS2322 ×5)

**File:** `packages/@nai/entitlement/src/index.ts`
**Root cause:** `getPlanEntitlements()` returns `EntitlementValue = string | number | boolean | string[] | null`. Assigning directly to typed fields (`instance_count: number`, `agents_enabled: string[]`, etc.) failed.
**Fix:** Added explicit type casts:
- `Number(planEnt['machine.instance.count'] ?? 1)`
- `String(planEnt['machine.model.tier'] ?? 'free')`
- `(planEnt['machine.agents.enabled'] as string[]) ?? []`
- `Boolean(planEnt['academy.pass'] ?? false)`

### Fix 5: `@nai/product-catalog` — `VALID_PLAN_IDS.includes()` type (TS2345 + TS7053)

**File:** `packages/product-catalog/src/validate.ts`
**Root cause:** `VALID_PLAN_IDS` is `as const` (readonly tuple of string literals). `.includes(plan.id: string)` rejects wider string type.
**Fix:** Created `VALID_PLAN_IDS_SET: ReadonlySet<string> = new Set(VALID_PLAN_IDS)` and used `.has(plan.id)` instead. Also cast `entitlements[plan.id as keyof typeof entitlements]`.

### Fix 6: `apps/auth` (auth-worker) — `.ts` imports + D1Session cast

**Files:** `apps/auth/src/{d1-audit-store.ts,index.ts}`
**Fix 1:** Removed `.ts` extensions from 4 relative imports.
**Fix 2:** `parseD1Session(d1Session)` → `parseD1Session(d1Session as unknown as Record<string, unknown>)` — `D1Session` interface doesn't match `Record<string, unknown>` signature but content is compatible.

### Fix 7: `@nai/policy-engine` — wrong `Entitlements` import source

**File:** `packages/@nai/policy-engine/src/{index.ts,test.ts}`
**Root cause:** Imported `type { Entitlements } from '@nai/entitlement'` but that type doesn't exist there. The package exports `ResolvedEntitlements` (structured: `{ machine, academy, custom }`), while `Entitlements` (flat record) lives in `@nai/product-catalog`. Policy-engine code uses `.machine`, `.academy`, `.custom` → needs `ResolvedEntitlements`.
**Fix:** `import type { ResolvedEntitlements as Entitlements } from '@nai/entitlement'` — alias import to preserve code using `Entitlements` name.

### Fix 8: `@nai/e2e-tests` — missing workers types + Session.plan_id

**File:** `tests/e2e/tsconfig.json`
**Fix:** Added `"types": ["@cloudflare/workers-types", "node"]` — needed for `Crypto` global used by `@nai/auth`.

**File:** `tests/e2e/package.json`
**Fix:** Added `@cloudflare/workers-types` to devDependencies.

**File:** `tests/e2e/src/p0b-e2e.ts`
**Fix:** Added `plan_id: 'nguyen-founder'` to Session literal — `Session` interface requires `plan_id` field (added in Phase 2).

### Fix 9: `@nai/api` — unused `Entitlements` import

**File:** `apps/api/src/index.ts`
**Fix:** Removed `type Entitlements` from `@nai/entitlement` import — was imported but never used in the API code.

## 6. Known Limitations (chưa phải production-ready)

### 6.1 InMemory stores (MVP)
- `@nai/evidence`, `@nai/compass`, `@nai/relic`, `@nai/approval`, `@nai/audit` use InMemory stores by default.
- Production needs D1/Postgres/Qdrant/R2 adapters (interfaces defined, not yet wired).
- Command resume/cancel requires client to send `paused_context` in body (no server-side persistence).

### 6.2 LLM provider
- `@nai/prism` GEN1 adapter configured but falls back to mock if `GEN1_GATEWAY_URL` not set.
- Real GEN1 calls not verified in this audit (no network calls made).

### 6.3 Console API wiring
- `apps/console` React islands call `http://localhost:8787` by default.
- Production needs `VITE_API_BASE=https://api.nguyenai.net` env var.
- No CORS verification done (API has CORS config but not tested cross-origin).

### 6.4 No E2E browser test
- P1-A E2E test exercises the full chain in-process (no HTTP, no browser).
- Production E2E needs real `fetch` against running Workers + browser flow.

### 6.5 Security audit not in scope
- This audit covers typecheck + build + test only.
- Security audit (CORS, CSP, auth, IDOR, injection) is separate per AGENTS.md.
