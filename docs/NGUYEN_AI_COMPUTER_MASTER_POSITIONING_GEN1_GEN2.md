## 2. What Nguyen AI Computer actually is

Nguyen AI is not a website with a few AI tools.

It is a **Personal AI Computer System on the cloud**.

Each user has their own machine:

```text
Nguyen AI Computer Instance
├── Identity & Ownership
├── Command Center
├── Model Mesh
├── Agent Team
├── Super Apps
├── Tool & Connector Kernel
├── Data Vault
├── Long-term Memory
├── Workflow Engine
├── Verification & Evidence
├── Approval Gates
├── Security Boundary
├── Cost Governor
├── Audit & Replay
├── Sync Layer
└── Self-Upgrade Registry
```

The machine can:

- accept commands in Vietnamese;
- plan autonomously;
- select the right model;
- distribute work to multiple Agents;
- call tools and connectors;
- execute long-running workflows;
- store memory;
- request approval before sensitive actions;
- verify results;
- store sources and evidence;
- report costs;
- recover from errors.

## 3. Not a chatbot

| Ordinary chatbot | Nguyen AI Computer |
|---|---|
| Answers single questions | Executes entire workflows |
| Uses one model | Model Mesh with multiple models |
| No private machine | Each user has a private instance |
| Limited memory | Structured long-term memory |
| No permissions | Permission and approval system |
| No full tools | Super Apps and connectors |
| No audit | Audit, evidence and replay |
| Not continuous | Scheduled and long-running tasks |

Example: a user does not just ask "write a business plan." They can command:

> "Research the market, analyze competitors, read all project documents, build a revenue model, prepare a bilingual pitch deck, save evidence and send me for approval before publishing."

Nguyen AI coordinates:

1. Researcher collects data.
2. Analyst analyzes.
3. Founder Agent plans.
4. Finance Agent builds model.
5. Verifier checks sources.
6. Formatter creates document.
7. Human Approval Gate requests user approval.
